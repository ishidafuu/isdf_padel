# task-status

## 概要

**タスク状況表示スキル** - タスク一覧と進捗状況を表示する際の標準形式

### 詳細参照

出力例・参考実装は [docs/concepts/tasks.md](../../docs/concepts/tasks.md) を参照。

## ステータスアイコン

| ステータス | アイコン | 説明 |
|-----------|---------|------|
| todo | ⬜ | 未着手 |
| in-progress | 🔄 | 進行中 |
| in-review | 👁️ | レビュー待ち |
| done | ✅ | 完了 |
| cancelled | ❌ | キャンセル |

## 優先度アイコン

| 優先度 | アイコン |
|-------|---------|
| high | 🔴 |
| medium | 🟡 |
| low | 🟢 |

## 実装方法

メイン Claude Code は Glob/Read ツールでタスクファイルを読み込み、この形式で表示する。

```
1. Glob("project/tasks/*/") でフォルダ一覧取得
2. Glob("project/tasks/2_in-progress/*.md") 等でファイル取得
3. Read でFrontmatter読み込み
4. 整形して表示
```

## 出力形式

```
===================================
タスク状況
===================================

🔄 🔴 [30101] ジャンプ機能実装 (game-dev)
⬜ 🟡 [30102] 敵キャラクターAI (game-dev)

===================================
統計
===================================
合計: 2
⬜ Todo: 1
🔄 In Progress: 1
```

## フィルタオプション

| オプション | 説明 |
|-----------|------|
| `--type game-dev` | タイプでフィルタ |
| `--status in-progress` | ステータスでフィルタ |
| `--priority high` | 優先度でフィルタ |
| `--all` | done/cancelled も含む |
| `--verbose` | 詳細表示（spec_ids, worktree等） |

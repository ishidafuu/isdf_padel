# Racket Trajectory Visual Specification

**Version**: 1.0.0  
**Status**: Draft  
**Last Updated**: 2026-02-08

## 概要

キャラクター表現よりも、ラケット軌道と接触タイミングの可視性を優先する。  
ラケットパーツはスイング中に専用軌道で上書きされ、接触点と現在位置を明確に表示する。

## Core Requirements (v0.7)

### REQ-31003-001: ラケット姿勢オーバーライド
**WHEN** スイングが有効な状態である  
**THE SYSTEM SHALL** ラケットパーツの位置・回転を軌道データで上書きする
- 通常キーフレームより接触駆動軌道を優先

---

### REQ-31003-002: 画面座標系への射影
**WHEN** ラケットの論理座標が更新される  
**THE SYSTEM SHALL** 既存の表示座標系（X=打ち合い方向, Y=Z+高さ）に投影して表示する

---

### REQ-31003-003: 接触タイミングの可視化
**THE SYSTEM SHALL** デバッグ表示で以下を描画できる
- 現在のラケット位置
- スイープ区間（前フレーム→現フレーム）
- 予測接触点

## 設計メモ

- 人型モーションの自然さより、物理的因果の理解しやすさを優先
- 将来的にキャラクターを抽象オブジェクトへ置換しても仕様を維持可能にする
